# Plan 03-02: Combat Engine and NPC Spawns

**Phase:** 03-character-and-combat
**Plan:** 2 of 3
**Scope:** CMBT-01, CMBT-02, CMBT-03, CMBT-04, CMBT-05, CMBT-06, CHAR-01

## Goal

Players can attack NPC monsters using D&D-lite dice mechanics with visible roll results. Combat resolves in ~2-second rounds. NPCs spawn from TOML data, have aggro behavior, and drop loot on death. Players who die respawn at their bind point with gear intact and an XP debt. The server sends Vitals messages after any state-changing action.

## Task 1: Combat Data Types, Protocol Messages, and Monster TOML Loader

### New Files

**`protocol/src/combat.rs`** — Combat-related protocol messages:

ClientMsg variants:
- `Attack { target: String }` — initiate or queue attack on target
- `Flee` — attempt to flee combat
- `UseAbility { ability_name: String }` — use class ability

ServerMsg variants:
- `Vitals { hp, max_hp, mana, max_mana, stamina, max_stamina, xp, level }` — pushed after state changes
- `CombatLog { entries: Vec<String> }` — dice roll results per round
- `CombatStart { combatants: Vec<String> }` — combat initiated
- `CombatEnd { result: String }` — combat finished (victory/flee/death)
- `ActionFail { reason: String }` — generic failure

**`server/src/combat/mod.rs`** — Combat module:
```rust
pub mod engine;
pub mod types;
pub mod manager;
```

**`server/src/combat/types.rs`** — Combat data types:
- `MonsterTemplate` — parsed from TOML: id, name, hp, ac, attack_bonus, damage_dice, damage_sides, damage_bonus, xp_value, aggro (passive/aggressive), loot table
- `LootEntry` — item template_id + drop chance percent, or gold range
- `ActiveMonster` — runtime instance: id, template_id, room_id, current_hp, max_hp
- `CombatInstance` — state machine for one fight: combatants, initiative order, queued actions, round count
- `CombatAction` — Attack(target), Flee, UseAbility(name), AutoAttack(target)
- `Combatant` — enum: Player { character_id, name } | Monster { instance_id, template_id, name }
- Dice helpers: `roll_d20()`, `roll_dice(count, sides)`, `DamageRoll` struct

**`server/src/combat/engine.rs`** — Pure combat resolution functions (no async, no DB):
- `roll_initiative(dex_mod: i32) -> i32` — d20 + dex mod
- `resolve_attack(attacker_bonus: i32, defender_ac: i32) -> AttackResult` — d20+bonus vs AC
- `roll_damage(count: u32, sides: u32, bonus: i32, is_crit: bool) -> (i32, String)` — returns (damage, description)
- `format_combat_log(attacker: &str, roll: i32, bonus: i32, ac: i32, hit: bool, damage: Option<i32>, description: &str) -> String` — "Grok rolled 14 + 2 STR vs AC 12 — HIT! 7 damage (1d6+3)"
- `AttackResult` — CriticalHit, Hit, Miss, CriticalMiss

**`server/src/combat/manager.rs`** — CombatManager that holds all active combats:
- `CombatManager` wraps `HashMap<RoomId, CombatInstance>`
- `start_combat(room_id, combatants) -> CombatInstance` — roll initiative, sort
- `queue_action(room_id, combatant_id, action)` — player queues their action for this round
- `tick() -> Vec<CombatEvent>` — process one round of all active combats: resolve actions in initiative order, apply damage, check deaths, return events to broadcast
- `CombatEvent` — enum of things to broadcast/send: LogEntries, VitalsUpdate, Death, CombatEnded, LootDropped

## Task 2: Combat Tick Loop, Command Handlers, Death/Respawn, and Vitals

### Modified Files

**`server/src/net/listener.rs`** — AppState gains:
- `combat_manager: Arc<RwLock<CombatManager>>`
- `monster_templates: Arc<HashMap<String, MonsterTemplate>>`
- `active_monsters: Arc<RwLock<HashMap<RoomId, Vec<ActiveMonster>>>>`

**`server/src/main.rs`** — Startup changes:
- Load monster templates from `world/data/monsters.toml`
- Load zone spawn tables and create initial monster instances
- Spawn combat tick task: `tokio::spawn(combat_tick_loop(...))`
- The tick loop runs every 2 seconds via `tokio::time::interval`

**`server/src/session/actor.rs`** — Extend dispatch and handlers:
- Add NS_COMBAT dispatch in `dispatch_frame`
- `handle_combat_message(msg)`:
  - `Attack { target }` — find target monster in room, start or join combat, queue attack action
  - `Flee` — queue flee action (50% success chance per attempt)
  - `UseAbility { ability_name }` — validate class ability, check cooldown, queue ability action
- After any state-changing world/combat action, send `Vitals` message (CHAR-01)
- On character death: reset HP to max, teleport to bind point, apply XP debt, send CombatEnd + Vitals

**`server/src/combat/manager.rs`** — `tick()` implementation:
1. For each active combat: iterate combatants in initiative order
2. If combatant has a queued action, execute it; otherwise auto-attack last target
3. `resolve_attack` + `roll_damage` for each attack
4. Accumulate combat log entries
5. Check for deaths: monsters (drop loot, award XP, start respawn timer) or players (death sequence)
6. If all monsters dead or all players fled, end combat
7. Return `Vec<CombatEvent>` for the caller to dispatch

**Combat tick task** (new function in main.rs or combat module):
```rust
async fn combat_tick_loop(
    combat_manager: Arc<RwLock<CombatManager>>,
    world: Arc<RwLock<World>>,
    room_channels: Arc<RwLock<RoomChannels>>,
    db: SqlitePool,
    // ... other state
) {
    let mut interval = tokio::time::interval(Duration::from_secs(2));
    loop {
        interval.tick().await;
        let events = combat_manager.write().await.tick(&world, &active_monsters).await;
        // Dispatch events: broadcast CombatLog to rooms, send Vitals to specific players, etc.
    }
}
```

**Monster spawning and respawn**:
- On startup: iterate zone room spawn tables, create ActiveMonster instances
- Track `respawn_timers: HashMap<(RoomId, String), Instant>` — when a monster dies, set timer
- In tick loop: check timers, respawn monsters whose timer expired

**Death sequence** (CMBT-06):
- Character HP reaches 0
- XP debt: subtract 10% of current XP (min XP = 0, no negative)
- Teleport character to bind point (default: starting_village:market_square)
- Restore HP to max_hp, mana to max_mana, stamina to max_stamina
- Persist: update character row in DB (xp, hp), update character_positions
- Send: CombatEnd { result: "You have been slain..." }, Vitals (restored), RoomDescription (bind point)
- Remove from combat instance
- Broadcast to old room: "{name} has been slain!"

**NPC aggro** (CMBT-04):
- When a player enters a room (via Move), check for aggressive monsters
- If aggressive monsters present and not already in combat, auto-start combat after 1-round delay
- Send CombatStart to the player, queue monster's first attack

**Vitals push** (CHAR-01):
- Send Vitals after: combat round (HP changed), equip/unequip (stats changed), character select, death/respawn
- Vitals message includes: hp, max_hp, mana, max_mana, stamina, max_stamina, xp, level
