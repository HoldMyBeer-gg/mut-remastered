# Plan 03-03: Inventory, Equipment, and Integration Tests

**Phase:** 03-character-and-combat
**Plan:** 3 of 3
**Scope:** CHAR-02, CHAR-03, CHAR-04, CHAR-05, CMBT-05

## Goal

Players can pick up items, drop them, equip gear to body slots, and see their stats change. Monsters drop loot on death. Players can write a biography. Integration tests validate all 16 Phase 3 requirements end-to-end.

## Prerequisites

- Plan 03-02 complete (combat engine, NPC spawns, death/respawn)
- All 50 existing tests pass

## Task 1: Item Templates, Inventory Protocol, and DB Schema

### New Files

**`world/data/items.toml`** — Item template data:
- Starter items + monster drops: rusty_shortsword, leather_armor, iron_ring, rat_tail, leather_scraps, spider_silk, bone_shard, old_shield, wooden_staff, simple_dagger, cloth_robe

**`server/src/inventory/mod.rs`** — Inventory module
**`server/src/inventory/types.rs`** — ItemTemplate, ItemKind (weapon/armor/accessory/junk), body slots
**`server/src/inventory/loader.rs`** — Load item templates from TOML
**`server/src/inventory/commands.rs`** — Inventory command handlers

### Protocol additions (extend world namespace)

Add to `protocol/src/world.rs` ClientMsg:
- `Inventory` — list carried items
- `GetItem { target: String }` — pick up from room floor
- `DropItem { target: String }` — drop to room floor
- `Equip { item_name: String }` — equip to appropriate slot
- `Unequip { slot: String }` — move equipped item to inventory
- `Stats` — view own character stats
- `Bio { text: String }` — set biography (CHAR-05)

Add to `protocol/src/world.rs` ServerMsg:
- `InventoryList { items: Vec<ItemInfo>, equipped: Vec<EquippedInfo> }`
- `GetItemOk { item_name: String }`
- `DropItemOk { item_name: String }`
- `EquipOk { item_name: String, slot: String }`
- `UnequipOk { slot: String, item_name: String }`
- `StatsResult { ... }`
- `BioOk`

### New Migration

**`server/migrations/007_inventory.sql`**:
- `items` table: id, character_id, template_id, slot (NULL=inventory, non-NULL=equipped)
- `room_items` table: id, room_id, template_id
- Add `gold` column to characters table

### Verification (Task 1)
- `cargo build` succeeds
- Item TOML parses correctly
- Migration SQL is valid
