# Plan 04-01: TUI Client with Split-Panel Layout and Network Connection

**Phase:** 04-native-tui-client  
**Plan:** 1 of 2
**Scope:** TUI-01, TUI-02, TUI-04

## Goal

The native TUI client connects to the MUT server, presents login/character selection screens, and shows the main game interface with split panels (room, minimap, game log, vitals, input). Terminal resize is handled gracefully.

## Task 1: Dependencies, Network Layer, and App State Machine

- Add ratatui, crossterm, tokio-tungstenite (NO — raw TCP) dependencies
- Build network module: connect, read_frame, write_frame (mirrors test helpers)
- App state machine: Connecting → Login → CharacterSelect → InGame
- Message routing: background reader task → mpsc → UI update

## Task 2: TUI Rendering and Input

- Login screen: username/password input fields
- Character select screen: list characters, create new, select
- Main game layout with Ratatui constraints
- Room description pane, game log (scrollable), vitals bar, input line
- Minimap/compass pane showing exits
- Terminal resize handling
- Input parsing: commands sent as protocol messages
