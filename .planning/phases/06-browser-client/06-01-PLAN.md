# Plan 06-01: WebSocket Server Endpoint and Browser Client

**Phase:** 06-browser-client
**Plan:** 1 of 1
**Scope:** NETW-02, NETW-03

## Goal

The game server handles WebSocket connections alongside existing TCP connections. A browser-based client using xterm.js connects via WebSocket and has feature parity with the native TUI client. For Phase 6 we build a minimal static HTML+JS client (SvelteKit armory deferred to v2).

## Task 1: Axum WebSocket Endpoint

- Add WebSocket upgrade handler in `server/src/net/listener.rs`
- New `server/src/net/ws.rs` — WebSocket connection handler
- WebSocket frames use the same length-prefixed binary protocol as TCP
- A WebSocket connection gets a ConnectionActor just like TCP

## Task 2: Static Web Client with xterm.js

- `web-client/index.html` — single page with xterm.js terminal
- `web-client/client.js` — WebSocket connection, protocol encode/decode, command input
- The web client sends the same binary protocol messages over WebSocket
- Display is text-based (ANSI formatting rendered by xterm.js)
- Server sends text-formatted responses that xterm.js renders directly
